Envio de trabalhos

Para enviar um trabalho, selecione a linguagem e os arquivos. Depois clique no botão "Enviar".

Os nome dos arquivos que serão enviados não devem ter acentos, espaços, tabulações, barras ou outros símbolos. Use apenas letras, dígitos, hifens, sublinhados e pontos no nome dos arquivos.

Os nomes dos arquivos devem ter sufixos de acordo com a linguagem:

  • C: .c ou .h
  • C++: .cpp ou .h
  • Fortran: .f ou .F
  • Java: .java
  • Octave: .m
  • PDF: .pdf
  • Pascal: .pas
  • Python3: .py

Para as linguagens C, C++, Java e Python3 a submissão pode ter um número qualquer de arquivos, a menos que haja uma restrição indicada no enunciado do trabalho.

  • Em Java, a função main deve estar em uma classe chamada Main; logo deve haver um arquivo chamado Main.java.
  • Em Python3, se houver mais de um arquivo então um deles deve se chamar main.py.

Para as linguagens Octave, Pascal e Fortran a submissão deve ter apenas um arquivo.

O programa será avaliado pela verificação do resultado da execução com alguns casos-de-teste. Cada caso-de-teste será executado redirecionando um arquivo para a entrada padrão e redirecionando a saída para outro arquivo, com um comando da forma 

programa <caso.in >execucao.out
e depois comparando a saída com a saída esperada usando diff 
diff execucao.out caso.out
e usando diff ignorando a formatação 
diff -q -b -B -i -w execucao.out caso.out
Se houver algum caso-de-teste aberto, o sqtpm exibirá a entrada, a saída produzida pelo programa e a saída esperada. Em alguns trabalhos a saída correta não é única. Nesses casos o sqtpm exibirá apenas a entrada e a saída produzida pelo programa para os casos-de-teste abertos.
